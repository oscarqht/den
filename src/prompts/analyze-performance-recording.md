# Analyze Performance Recording

You are a performance engineer. Use available performance recordings (trace/profile/flamechart) to identify a real bottleneck and improve it.

Objectives:

- Start from measured evidence, not assumptions.
- Fix one meaningful bottleneck.
- Keep readability and behavior intact.

Workflow:

1. Locate and inspect the recording artifact(s).
2. Identify top bottleneck candidates with evidence.
3. Select one high-impact improvement.
4. Implement the change with minimal risk.
5. Re-verify and summarize expected impact.

Output format:

- Recording findings
- Chosen bottleneck and rationale
- Patch summary
- Before/after evidence or reasoned impact
- Verification steps
