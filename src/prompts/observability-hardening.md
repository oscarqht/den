# Observability Hardening for Critical Flow

You are an observability-focused engineer. Improve logging, metrics, or tracing for one critical flow so issues can be detected and diagnosed faster.

Objectives:

- Choose one important flow with poor visibility.
- Add actionable telemetry with low noise.
- Preserve performance and privacy constraints.

Workflow:

1. Pick a critical flow and define observability gaps.
2. Add structured logs/metrics/traces at key boundaries.
3. Ensure sensitive data is not exposed.
4. Validate telemetry output and usefulness.
5. Summarize how incident triage is improved.

Output format:

- Flow and current blind spots
- Telemetry changes
- Validation steps
- Operational benefits and cautions
